In an era where SaaS dominates, we made a deliberate decision to build iqtoolkit.ai as **self-hosted only**. No cloud version. No managed service. No vendor lock-in. Ever.

Here's why.

## The Problem with Cloud-Only Database Tools

Most database management tools today are cloud-based SaaS products. While convenient, this approach creates serious concerns for enterprises:

### Security & Privacy Risks

- **Data Exposure**: Your database schemas, queries, and metadata are sent to external services
- **Compliance Nightmares**: HIPAA, PCI DSS, and GDPR regulations often prohibit sending data externally
- **Loss of Control**: You don't know how third parties process or store your sensitive information

### Operational Concerns

- **Vendor Lock-In**: Changing providers means migration pain and data export challenges
- **Cost Escalation**: Per-seat or per-query pricing can become expensive at scale
- **Availability Dependency**: Your operations depend on someone else's uptime SLA

## Our Solution: Self-Hosted Everything

iqtoolkit.ai runs entirely on **your infrastructure**. Whether that's Docker on a single machine, Kubernetes clusters, or on-premises data centers.

### Complete Data Privacy

```bash
# Your data never leaves your network
docker run -d iqtoolkit/iqtoolkit:latest
# ✓ All analysis happens locally
# ✓ No external API calls (unless you choose OpenAI)
# ✓ Zero data exposure
```

Your database schemas, queries, and performance metrics remain 100% private.

### Regulatory Compliance Made Easy

Self-hosted deployment makes it straightforward to comply with regulations:

- **HIPAA**: Healthcare data stays within your protected environment
- **PCI DSS**: Payment card data never leaves your secure network
- **GDPR**: European user data remains in your EU servers
- **SOC 2**: Maintain complete control over sensitive information

### Predictable, Transparent Costs

No per-request charges. No surprise bills. Just:

- **One-Time Setup**: Install once via Docker or Helm
- **Unlimited Usage**: Analyze as many databases as you need
- **Scalable**: Add hardware instead of paying per user/query
- **Open Source**: MIT/Apache license means no licensing fees

## AI That Respects Your Privacy

We support **two AI options** - you choose based on your security requirements:

### Ollama (Default - Private)

Run AI models entirely on-premises:

```bash
# Install Ollama
docker run -d ollama/ollama

# Download model (one-time)
ollama pull llama2

# Configure iqtoolkit
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=llama2
```

**Benefits:**

- Zero external API calls
- Perfect for regulated industries
- No per-request costs
- Works offline

### OpenAI (Optional - Cloud)

For teams that prefer cloud AI:

```bash
# Configure with your API key
OPENAI_API_KEY=your-key
OPENAI_MODEL=gpt-4
```

**Benefits:**

- Latest AI capabilities
- Powerful analysis
- Managed infrastructure
- You still control when/what to send

The choice is yours. Both are first-class supported options.

## Deployment Options

### Docker (Development & Small Teams)

```bash
docker-compose up -d
```

Perfect for:

- Local development
- Small team deployments
- Quick testing
- Single-server setups

### Kubernetes with Helm (Enterprise)

```bash
helm install iqtoolkit iqtoolkit/iqtoolkit \
  --set deployment.replicas=3 \
  --set ollama.enabled=true
```

Designed for:

- Production environments
- Multi-environment setups (dev/staging/prod)
- High availability
- Enterprise scale

Works on:

- AWS EKS
- Google GKE
- Azure AKS
- Self-managed Kubernetes
- On-premises K8s

### On-Premises (Complete Air-Gap)

Install without internet connectivity:

- Download Docker images offline
- Transfer to secure network
- Deploy behind firewall
- Perfect for government, defense, finance
